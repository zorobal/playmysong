# PlayMySong - État des Lieux et Prochaines Étapes

**Date:** 11/03/2026  
**Projet:** PlayMySong - Application de demande de chansons en établissement

---

## ✅ Ce qui a été fait

### 1. Infrastructure & Déploiement
- **Frontend:** Déployé sur Firebase Hosting (https://playmysong-998d4.web.app)
- **Backend:** Déployé sur Vercel (https://playmysong-mu.vercel.app)
- **Base de données:** PostgreSQL sur Neon.tech

### 2. Authentification & Rôles
| Rôle | Description |
|-------|-------------|
| SUPER_ADMIN | Gestion de tous les établissements |
| ADMIN | Gestion d'un établissement (utilisateurs, playlists, demandes) |
| USER | Utilisateur d'un établissement (peut voir les demandes) |
| CLIENT | Client qui scanne le QR Code (une demande par scan) |

### 3. QR Code
- Génération automatique du QR Code par établissement
- URL: `https://playmysong-998d4.web.app/client?establishmentId=XXX`
- Affichage dans le SuperAdmin Dashboard avec bouton de téléchargement

### 4. Page Client (ClientPage.jsx)
- Affichage des playlists de l'établissement
- Recherche YouTube (si API configurée)
- Sélection d'une chanson
- Ajout optionnel d'un message
- Ajout optionnel d'un selfie
- **Une seule demande par scan** (stocké en localStorage)
- Page de remerciement avec les détails de la demande

### 5. Upload de musique (Admin)
- Modal pour ajouter des chansons aux playlists
- Extraction automatique du titre depuis le nom du fichier
- Possibilité de modifier titre, artiste, chemin de fichier, YouTube ID

### 6. Flux des demandes
```
Client scanne QR → Page Client → Sélection chanson → Message + Selfie (optionnel) 
→ Envoi → Demande en attente (PENDING)
→ Admin/User valide → Demande validée (VALIDATED) → File d'attente
→ Admin/User rejette → Demande rejetée (REJECTED)
```
