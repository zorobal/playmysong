# PlayMySong - Historique des Modifications

**Date:** 13/03/2026  
**Projet:** PlayMySong - Application de demande de chansons en établissement

---

## 📜 Historique des Corrections

### 2026-03-13 - Problème: Boutons Valider/Rejeter ne fonctionnent pas sur User Dashboard

**Problème:** Les boutons de validation et de rejet ne fonctionnaient pas sur la page User Dashboard.

**Solution:** 
- Ajout des événements Socket.IO pour émettre `request_validated` et `request_rejected` après chaque action
- Backend: `backend/src/api/index.js` - Les endpoints `/api/request/:id/validate` et `/api/request/:id/reject` émettent maintenant des événements socket

---

### 2026-03-13 - Problème: NowPlaying - La vidéo s'arrête après quelques secondes

**Problème:** Le lecteur YouTube se réinitialisait toutes les 10 secondes lors du refresh, causant l'arrêt de la vidéo.

**Solution:**
- Comparaison du videoId avant re-render pour éviter de recréer le lecteur
- mise à jour partielle de la page (queue seulement) sans toucher au lecteur
- Socket.IO ajouté pour recevoir les mises à jour en temps réel
- Fichier: `frontend/public/nowplaying.html`

---

### 2026-03-13 - Problème: File d'attente garde les chansons même après lecture

**Problème:** Les chansons jouées restaient affichées dans la file d'attente.

**Solution:**
- Ajout du endpoint `/api/request/:id/complete` pour marquer les chansons comme COMPLETED
- Socket event `request_completed` émis après chaque chanson terminée
- User Dashboard: filtre pour afficher seulement PENDING dans "Demandes" et VALIDATED dans "File d'attente"
- Auto-suppression des chansons terminées de la queue

---

### 2026-03-13 - Problème: Client ne peut pas faire nouvelle demande sans rescanner

**Problème:** Un client qui a déjà fait une demande ne pouvait pas en faire une nouvelle sans rescanner le QR Code.

**Solution:**
- Modification de ClientPage.jsx: le bouton "Nouvelle demande" remplacé par "Scanner QR Code"
- Redirection vers `/clienthome` avec instructions pour scanner à nouveau
- Suppression du lien de connexion en bas de page

---

### 2026-03-13 - Problème: Écran d'attente NowPlaying pas convivial

**Problème:** Quand aucune chanson n'est en attente, l'écran restait figé sur la dernière chanson jouée.

**Solution:** Ajout d'un écran d'attente moderne avec:
- Message "En attente de demandes"
- Animation de pulsation
- Instructions pour scanner le QR Code
- Fichier: `frontend/public/nowplaying.html`

---

### 2026-03-13 - Fonctionnalité: Playlist Établissement (Lecture Auto)

**Problème:** Besoin d'une playlist qui se joue automatiquement quand la file d'attente est vide.

**Solution:** Implémentation complète:
1. **Backend:** 
   - Endpoint `/api/request/playlist/current` retourne maintenant `establishmentPlaylist`
   - Endpoint `/api/request/playlist/establishment/next` pour récupérer la prochaine chanson
   - Fichier: `backend/src/api/index.js`

2. **Frontend Admin:**
   - Ajout d'un nouvel onglet "🎼 Playlist Établissement"
   - Interface pour ajouter/supprimer des chansons YouTube
   - Fichier: `frontend/src/pages/AdminDashboard.jsx`

3. **NowPlaying:**
   - Joue automatiquement la playlist quand queue vide
   - Bascule vers la file d'attente dès qu'une chanson validée arrive
   - Badge affichant la source (Playlist Établissement vs Demande client)
   - Lecture en boucle de la playlist

---

### 2026-03-13 - Fonctionnalité: URL de redirection pour QR Code

**Problème:** Le QR Code affichait une URL longue visible.

**Solution:**
- Création d'une page de redirection: `frontend/src/pages/RedirectPage.jsx`
- Nouvelle route `/r/:id` qui redirige automatiquement vers `/client?establishmentId=XXX`
- QR Code mis à jour: `https://playmysong-998d4.web.app/r/ID` (plus court)
- Fichiers modifiés:
  - `frontend/src/App.jsx`
  - `frontend/src/pages/RedirectPage.jsx`
  - `backend/src/api/index.js`
  - `backend/src/services/qrcodeService.js`

---

### 2026-03-13 - Problème: Playlist Établissement ne joue pas en boucle

**Problème:** Les chansons de la playlist se relançaient sur la première chanson.

**Solution:**
- Correction de la logique d'index dans `playNext()`
- Recherche de l'index actuel par youtubeId pour continuer correctement

### Flux Client
```
QR Code → /r/ID (redirect) → /client?establishmentId=XXX
  ↓
Page Client (onglets Playlist | YouTube)
  ↓
Sélection chanson + Message + Selfie (optionnel)
  ↓
POST /requests (stocké en localStorage)
  ↓
Admin/Operator Dashboard → Valider/Rejeter
  ↓
NowPlaying → Affichage + Lecture auto
```

### Flux Playlist Établissement
```
Admin crée playlist → Ajoute chansons YouTube
  ↓
NowPlaying vérifie: File d'attente vide?
  ↓
OUI → Joue Playlist Établissement en boucle
  ↓
NON → Joue File d'attente (priorité haute)
  ↓
Chanson validée arrive → Bascule vers file d'attente
```

## 🔴 Problème Non Résolu - Sécurité QR Code

### Le Problème
L'URL du QR Code (même courte `/r/ID`) est visible lors du scan et peut être copiée/partagée.

### Solutions Possibles

1. **Application Mobile Native** (Recommandé)
   - QR Code avec URL scheme: `playmysong://etablissement/ID`
   - Scan ouvre l'app directement
   - URL complètement cachée
   - Meilleure expérience utilisateur

2. **Firebase Dynamic Links** (Déprécié)
   - Bientôt supprimé par Google
   - Non recommandé
